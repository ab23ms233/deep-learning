TMDB Movie Recommendation Project - Handoff Document
Project Overview

The goal of this project is to build a content-based movie recommendation system using data obtained from the TMDB API.

The recommendation system currently uses movie plot summaries (overviews) and TF-IDF vectorization to compute movie similarity and recommend movies with similar content.

The project is intended as an NLP learning project and is being developed by a student learning Natural Language Processing and Machine Learning.

Step 3: TF-IDF Vectorization

Current vectorization:

from sklearn.feature_extraction.text import TfidfVectorizer

tfidf = TfidfVectorizer(
    stop_words="english"
)

tfidf_matrix = tfidf.fit_transform(df["overview"])
Step 4: Similarity Matrix

Current implementation:

from sklearn.metrics.pairwise import cosine_similarity

sim_matrix = cosine_similarity(
    tfidf_matrix,
    tfidf_matrix
)

Result:

sim_matrix[i][j]

represents similarity between movie i and movie j.

Step 5: Recommendation Generation

Current approach:

Find movie index.
Extract similarity scores.
Sort descending.
Return top N movies.

Current code:

sim_scores = list(enumerate(sim_matrix[index]))

sorted_scores = sorted(
    sim_scores,
    key=lambda x: x[1],
    reverse=True
)

top_indices = [
    i[0]
    for i in sorted_scores[:num_recommendations+1]
]

The queried movie is then removed from the recommendation list.

Genre-Based Evaluation

Current evaluation uses Jaccard similarity between genres.

Function:

def genre_similarity(query_genre, recommended_genre):
    q_set = set(query_genre)
    g_set = set(recommended_genre)

    return len(q_set & g_set) / len(q_set | g_set)

This is Jaccard similarity:

Intersection / Union

Example:

A = {Action, Crime, Thriller}
B = {Action, Crime}

Score:

2 / 3 = 0.667

This is preferred over:

Intersection / len(A)

or

Intersection / len(B)

because Jaccard treats both sets symmetrically.

Alternative Advanced Approach

Instead of mixing everything into text:

Create separate feature matrices:

overview_tfidf
genre_matrix
keyword_matrix

Then combine:

from scipy.sparse import hstack

final_features = hstack([
    overview_tfidf,
    genre_matrix,
    keyword_matrix
])

Then compute:

cosine_similarity(
    final_features,
    final_features
)

This approach is expected to outperform simple concatenation.

Discussion About IDs vs Names

Question:

Should genre IDs and keyword IDs be used instead of names?

Example:

28
80
53

Answer:

No.

Reason:

These numbers are arbitrary identifiers.

TF-IDF treats:

action crime thriller

and

28 80 53

similarly as tokens.

Using names is preferable because:

interpretable
easier debugging
no loss of information
Keyword Extraction Bottleneck

Problem:

Fetching keywords requires one API request per movie.

Example:

/movie/{id}/keywords

This becomes slow for thousands of movies.

Suggested solutions:

Use overview only.
Use overview + genres.
Fetch keywords only for top/popular movies.
Save fetched keywords locally.
Use parallel requests.
Use Kaggle TMDB datasets containing keywords.

Recommendation:

Do not spend excessive effort fetching keywords for every movie.

Focus on the NLP pipeline.
